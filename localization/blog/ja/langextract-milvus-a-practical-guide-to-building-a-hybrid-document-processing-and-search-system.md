<p><a href="https://milvus.io/blog/why-im-against-claude-codes-grep-only-retrieval-it-just-burns-too-many-tokens.md">以前のブログで</a>、多くのコーディングエージェントにおけるコード検索への2つの一般的なアプローチを比較した：</p>
<ul>
<li><p><strong>ベクトル検索を利用したRAG（セマンティック検索）</strong>- Cursorのようなツールで利用されている。</p></li>
<li><p><code translate="no">grep</code> <strong> （リテラル文字列マッチング</strong><strong>）によるキーワード検索</strong>- Claude CodeやGeminiで使用されています。</p></li>
</ul>
<p>この投稿は多くのフィードバックを呼び起こした。<code translate="no">grep</code> 、しばしば無関係なマッチが含まれ、コンテキストが肥大化することを指摘し、RAGを主張する開発者もいた。また、精度がすべてであり、エンベッディングを信頼するにはまだあいまいすぎると、キーワード検索を擁護する人もいた。</p>
<p>どちらの意見にも一理ある。現実には、完璧で万能なソリューションは存在しない。</p>
<ul>
<li><p>エンベッディングだけに頼れば、厳密なルールや完全一致を見逃すことになる。</p></li>
<li><p>キーワードだけに頼れば、コード（またはテキスト）が実際に何を意味するのか、意味論的な理解を失うことになります。</p></li>
</ul>
<p>このチュートリアルでは、<strong>両方のアプローチをスマートに組み合わせる</strong>方法を紹介します。<a href="https://github.com/google/langextract">LangExtract（</a>LLMを使用して、乱雑なテキストを正確なソース帰属を持つ構造化データに変換するPythonライブラリ）を、オープンソースの高性能ベクトルデータベースである<a href="https://milvus.io/">Milvusと共に</a>使用して、よりインテリジェントで高品質な文書処理・検索システムを構築する方法を紹介します。</p>
<h3 id="Key-Technologies-We’ll-Use" class="common-anchor-header">使用する主要技術</h3><p>この文書処理・検索システムの構築を始める前に、このチュートリアルで使用する主要技術を見てみましょう。</p>
<h3 id="What-is-LangExtract" class="common-anchor-header">LangExtractとは？</h3><p><a href="https://github.com/langextract/langextract">LangExtractは</a>、Googleによってオープンソース化された新しいPythonライブラリで、LLMを利用して、厄介な非構造化テキストをソースの帰属を伴う構造化データに変換します。情報抽出のようなタスクを非常にシンプルにするため、すでに人気があります（13K+ GitHub stars）。</p>
<p>
  <span class="img-wrapper">
    <img translate="no" src="https://assets.zilliz.com/1_c04bdf275b.png" alt="" class="doc-image" id="" />
    <span></span>
  </span>
主な機能は以下の通り：</p>
<ul>
<li><p>構造化抽出：スキーマを定義し、名前、日付、場所、料金、その他の関連情報を抽出します。</p></li>
<li><p>ソースのトレーサビリティ：抽出されたすべてのフィールドは元のテキストにリンクされ、幻覚の可能性を低減します。</p></li>
<li><p>長い文書にも対応：チャンキング＋マルチスレッドで数百万文字を処理します。</p></li>
</ul>
<p>
  <span class="img-wrapper">
    <img translate="no" src="https://assets.zilliz.com/2_6a4b42a265.gif" alt="" class="doc-image" id="" />
    <span></span>
  </span>
</p>
<p>LangExtractは、法律、医療、法医学など、精度が重要な分野で特に役立ちます。例えば、RAGで巨大なテキストブロックを取得する代わりに、LangExtractは、意味的な文脈を保持したまま、気になる日付、句、患者の属性だけを抽出することができます。</p>
<h3 id="What’s-Milvus" class="common-anchor-header">Milvusとは？</h3><p><a href="https://milvus.io/">Milvusは</a>オープンソースのベクトルデータベースで、Github上で36K以上のスターを持ち、様々な業界で10K以上の企業で採用されています。Milvusは、RAGシステム、AIエージェント、レコメンデーションエンジン、異常検知、セマンティック検索などで広く使用されており、AIを活用したアプリケーションの中核となるビルディングブロックとなっています。</p>

<ul>

</ul>
